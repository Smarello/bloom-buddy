## User Story

**Epic:** EP-002 — Caricamento e Validazione Foto
**Priorità:** MEDIUM | **Story Points:** 2

**Story**
As Marco,
I want to drag and drop a photo of my plant into the upload area when using the app on my desktop,
so that uploading is quick and intuitive from my work computer.

**Criteri di Accettazione**
- [ ] Area di drop visibile e riconoscibile su desktop
- [ ] Feedback visivo durante il trascinamento del file sull'area (evidenziazione bordo/sfondo)
- [ ] File rilasciato viene validato e compresso come da upload standard
- [ ] Messaggio d'errore chiaro se il file trascinato non è un'immagine supportata

---

## Soluzione Tecnica

La funzionalità di drop è già parzialmente presente in `UploadZone` (`onDrop`, `onDragOver`), ma manca completamente il feedback visivo durante il trascinamento. La soluzione introduce uno stato booleano locale `isDragOver` gestito da `onDragEnter` e `onDragLeave`/`onDrop`, con un contatore ref (`dragEnterCounterRef`) per evitare i falsi "leave" causati dal cursore che attraversa elementi figli. Il feedback visivo si applica tramite classi Tailwind condizionali sull'area di drop, senza modifiche al dominio (`useImageUpload` o la logica di validazione/compressione).

- Aggiungere `dragEnterCounterRef` (useRef) e `isDragOver` (useState) in `UploadZone` per tracciare il drag in modo affidabile
- Gestire `onDragEnter` (incrementa counter, imposta isDragOver=true) e aggiornare `onDragLeave` (decrementa, se counter=0 imposta isDragOver=false) e `onDrop` (reset counter, isDragOver=false, poi logica drop esistente)
- Applicare classi CSS condizionali sulla zona: bordo più marcato (es. `border-primary-500`) e sfondo leggermente evidenziato durante `isDragOver=true`, mantenendo lo stile esistente altrimenti

## Strategia di Test

I test verificano il comportamento degli eventi drag tramite `fireEvent.dragEnter`, `fireEvent.dragLeave`, `fireEvent.drop` di React Testing Library, controllando che le classi CSS cambino correttamente e che il file droppato sia passato a `gestisciSelezioneFile`.

- Test: `dragEnter` sulla zona imposta la classe di evidenziazione (isDragOver visibile nel DOM)
- Test: `dragLeave` dalla zona rimuove la classe di evidenziazione
- Test: `drop` con file valido chiama `gestisciSelezioneFile` con il file corretto e rimuove la classe di evidenziazione
- Test: `drop` con file non valido chiama `gestisciSelezioneFile` (la validazione gestita dall'hook mostra l'errore)
- Test: sequenza `dragEnter` figlio → `dragLeave` figlio non resetta isDragOver (robustezza contro bubble degli eventi)
