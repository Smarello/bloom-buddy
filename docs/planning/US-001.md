## Soluzione Tecnica

Lo scaffolding usa `create-next-app` con App Router e TypeScript, poi configura Tailwind CSS 4 (configurazione CSS-based, non più file JS/TS) e shadcn/ui. Il design system dei mockup esistenti (`docs/mockups/bloom-buddy-app/shared.css`) viene tradotto in variabili CSS custom e nella configurazione Tailwind/shadcn, mantenendo l'estetica "Organic Greenhouse" con palette sage green/terracotta, font Quicksand (display) e Nunito (body), e texture organiche.

- **Struttura progetto**: segue il layout definito nel PRD (`src/app/`, `src/components/`, `src/lib/`, `src/types/`) con organizzazione feature-based
- **Design system**: i design token del mockup (colori, tipografia, spacing, radius, shadows, animazioni) vengono integrati in `src/app/globals.css` come variabili CSS e nella configurazione di Tailwind 4, rendendo il tema coerente con i mockup
- **Homepage**: hero section con branding (logo SVG custom, titolo con gradient, sottotitolo), upload zone come placeholder (non funzionale in questa story), sezione "Come funziona" con 3 step — tutto riprodotto fedelmente dai mockup
- **Componenti base**: navbar sticky con backdrop-filter, footer con branding, struttura layout root con font Google (Quicksand + Nunito)
- **Deploy**: configurazione `vercel.json` minimale (o zero-config), collegamento repo GitHub a Vercel

---

## Strategia di Test

Focus su rendering corretto e configurazione pulita, essendo il primo scaffolding del progetto.

- **Build test**: `npm run build` deve completare senza errori TypeScript o lint
- **Lint & type check**: ESLint e `tsc --noEmit` passano senza warning
- **Visual check manuale**: la homepage si visualizza correttamente su viewport mobile (375px) e desktop (1280px), con layout responsive coerente ai mockup
