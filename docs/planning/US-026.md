## User Story

**Epic:** EP-010 — Evoluzione Collezione con Identità Propria
**Priorità:** HIGH | **Story Points:** 2

**Story**
As Marco,
I want poter rinominare le mie collezioni con nomi personalizzati,
so that posso distinguere piante della stessa specie (es. "Basilico balcone" e "Basilico cucina").

**Criteri di Accettazione**

- [ ] Nella pagina dettaglio collezione il nome è modificabile (click per editare o pulsante di modifica)
- [ ] Il nome viene validato: non vuoto, lunghezza massima ragionevole (es. 100 caratteri)
- [ ] Il salvataggio avviene tramite API che aggiorna il campo `nome` della Collezione
- [ ] Due collezioni possono avere lo stesso nome (nessun vincolo di unicità sul nome)
- [ ] Il nome aggiornato si riflette nella pagina principale della collezione senza ricaricare

---

## Soluzione Tecnica

La soluzione prevede una nuova API route PATCH su `/api/collezione/[idCollezione]` per aggiornare il nome, e un componente client dedicato per l'intestazione della pagina dettaglio che gestisce l'editing inline con aggiornamento ottimistico.

- **API PATCH `/api/collezione/[idCollezione]`**: riceve `{ nome: string }`, valida (non vuoto, max 100 chars), verifica ownership tramite sessione, aggiorna via Prisma e restituisce il nome aggiornato. La route file `src/app/api/collezione/[idCollezione]/route.ts` esiste già — si aggiunge l'export `PATCH` accanto alla logica esistente.
- **Componente client `IntestazioneCollezione`**: estratto dalla pagina dettaglio, gestisce lo stato di editing (click su pulsante matita → input inline → conferma/annulla). Usa `fetch` per chiamare l'API PATCH e aggiorna il titolo ottimisticamente.
- **Pagina dettaglio**: resta Server Component, passa `id`, `nome` e `nomeScientifico` al nuovo componente client per l'intestazione.

## Strategia di Test

Focus su validazione API (input invalidi, autorizzazione, happy path) e verifica del flusso UI di editing inline.

- Test API: happy path (nome aggiornato), nome vuoto (400), nome troppo lungo (400), collezione non propria (403), collezione inesistente (404)
- Test componente: rendering pulsante modifica, transizione a modalità editing, salvataggio e ritorno a visualizzazione
