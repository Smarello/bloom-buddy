## User Story

**Epic:** EP-006 — Collezione Personale e Storico Analisi
**Priorità:** HIGH | **Story Points:** 3

**Story**
Come Giulia, voglio sfogliare le mie piante salvate, così da poter consultare i consigli di cura e lo storico in qualsiasi momento.

**Criteri di Accettazione**

- [ ] La pagina mostra tutte le piante salvate dall'utente autenticato, ordinate per data di salvataggio (più recente prima)
- [ ] Ogni pianta mostra: foto thumbnail, nome comune, nome scientifico e data dell'ultima analisi
- [ ] Se nessuna pianta è salvata, la pagina mostra un messaggio incoraggiante con CTA per avviare una nuova analisi
- [ ] La pagina non è accessibile senza login (reindirizza alla pagina di login)

---

## Soluzione Tecnica

La pagina `/collezione` sarà un Server Component che verifica la sessione e fa la query Prisma direttamente, senza necessità di un endpoint API GET dedicato. Un componente client `SchedaPiantaCollezione` renderizza ogni card con thumbnail, nomi e data. L'empty state usa il pattern hero centrato con icona e CTA verso la home. La navbar viene aggiornata con un link "La mia collezione" visibile solo per utenti autenticati.

- **Pagina server-rendered** con `ottieniSessioneServer()` + `redirect('/accesso')` se non autenticato
- **Query diretta** `prisma.analisi.findMany({ where: { utenteId }, orderBy: { createdAt: 'desc' } })` — cast `datiAnalisi` a `PlantAnalysis`
- **Componente card** con thumbnail (Next.js `Image`), nome comune, nome scientifico, data formattata, indicatore salute
- **Link navbar** per utenti autenticati, posizionato prima dell'avatar

---

## Strategia di Test

Test focalizzati sulla protezione della pagina, il rendering corretto della lista e l'empty state.

- Test redirect a `/accesso` quando l'utente non è autenticato
- Test rendering lista piante con dati mock (verifica thumbnail, nomi, data)
- Test empty state quando la collezione è vuota (verifica messaggio e CTA)
