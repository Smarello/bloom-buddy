## User Story

**Epic:** EP-002 — Caricamento e Validazione Foto
**Priorità:** HIGH | **Story Points:** 3

**Story**
As Giulia,
I want to upload a photo of my plant from my phone gallery or take a new one with the camera and see a preview,
so that I can get my plant analyzed without any technical knowledge.

**Criteri di Accettazione**

- [ ] File picker nativo del browser per selezione da galleria
- [ ] Attributo capture per scatto diretto da fotocamera su mobile
- [ ] Anteprima dell'immagine selezionata visibile all'utente
- [ ] Compressione automatica dell'immagine nel browser prima dell'invio
- [ ] Validazione formato file (JPEG, PNG, WebP) con messaggio d'errore chiaro se il formato non è supportato
- [ ] Validazione dimensione massima file con messaggio d'errore chiaro se il file è troppo grande

---

## Soluzione Tecnica

L'approccio è trasformare il componente `UploadZone` (attualmente un placeholder statico) in un componente client interattivo con gestione stato completa. La logica di selezione, validazione, compressione e preview viene incapsulata in un hook custom `useImageUpload` per separare la business logic dalla presentazione.

- **Hook `useImageUpload`**: gestisce l'intero ciclo di vita dell'immagine — selezione file → validazione formato/dimensione → compressione con `browser-image-compression` → generazione URL preview con `URL.createObjectURL` → cleanup. Espone stato (`fileSelezionato`, `urlAnteprima`, `statoCompressione`, `erroreValidazione`) e azioni (`selezionaFile`, `rimuoviFile`).
- **Input file hidden duplice**: un `<input type="file" accept="image/jpeg,image/png,image/webp">` per la galleria, un secondo con attributo `capture="environment"` per lo scatto da fotocamera su mobile. I pulsanti esistenti nell'UploadZone diventano trigger per i rispettivi input.
- **Costanti di validazione** (`FORMATI_IMMAGINE_ACCETTATI`, `DIMENSIONE_MASSIMA_FILE_BYTE`) in un modulo `src/lib/image/costanti-validazione.ts` riusabile lato server in US-006.
- **Compressione client-side** con `browser-image-compression`: riduzione a max 1MB / 1920px lato lungo prima dell'invio, per ridurre latenza upload e consumo token API.
- **Preview con stati UI**: dopo la selezione, l'UploadZone mostra l'anteprima dell'immagine con opzione di rimuovere e ricaricare. Durante la compressione, un indicatore visivo informa l'utente. In caso di errore, un messaggio chiaro con icona e testo descrittivo.

## Strategia di Test

La strategia si concentra su test unitari del hook `useImageUpload` e test di integrazione del componente `UploadZone` per verificare i flussi utente critici: selezione valida, rifiuto formato, rifiuto dimensione, rendering preview.

- **Test unitari hook `useImageUpload`**: validazione formati accettati/rifiutati, validazione dimensione sopra/sotto soglia, stato errore corretto per ogni caso, cleanup URL preview su rimozione file
- **Test componente `UploadZone`**: rendering stato iniziale con pulsanti gallery/camera, rendering anteprima dopo selezione file valido, rendering messaggio errore per formato non supportato, rendering messaggio errore per file troppo grande, accessibilità (aria-label, ruoli)
- **Strumenti**: Vitest + React Testing Library, mock di `browser-image-compression` e `URL.createObjectURL`
