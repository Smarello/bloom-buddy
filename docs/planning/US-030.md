## User Story

**Epic:** EP-010 — Evoluzione Collezione con Identità Propria
**Priorità:** HIGH | **Story Points:** 3

**Story**
As Giulia, I want un pulsante nella pagina della mia collezione per fare una nuova analisi direttamente collegata a quella collezione, so that posso monitorare la stessa pianta nel tempo senza dover riassociare manualmente l'analisi.

**Criteri di Accettazione**

- [ ] Nella pagina dettaglio collezione è presente un pulsante/CTA "Nuova analisi" ben visibile
- [ ] Cliccando il pulsante si apre il flusso di upload foto con il contesto della collezione (collezioneId passato come parametro)
- [ ] L'analisi completata viene salvata con il `collezioneId` della collezione di partenza
- [ ] Dopo il salvataggio, l'utente viene reindirizzato alla pagina della collezione dove vede la nuova analisi in cima alla lista
- [ ] Se l'analisi fallisce o viene annullata, la collezione resta invariata

---

## Soluzione Tecnica

Si aggiunge il supporto per il `collezioneId` opzionale lungo tutta la catena: pagina dettaglio → homepage con query param → hook useAnalysis → sessionStorage → pagina /analysis → AnalysisResult → POST API. L'API viene modificata per accettare un `collezioneId` nel FormData: se presente, verifica ownership e aggiunge l'analisi alla collezione esistente; se assente, mantiene il comportamento attuale (crea nuova collezione).

- **API**: POST `/api/collezione` accetta campo opzionale `collezioneId` nel FormData. Se presente: verifica che la collezione esista e appartenga all'utente, poi crea solo l'analisi con quel `collezioneId`. Se assente: crea nuova collezione + analisi (comportamento attuale)
- **Pagina dettaglio**: aggiunge CTA "Nuova analisi" nell'intestazione che naviga a `/?collezioneId={id}`
- **UploadZone**: legge `collezioneId` dai search params dell'URL, lo include nei dati salvati in sessionStorage tramite `useAnalysis`
- **useAnalysis hook**: accetta `collezioneId` opzionale e lo persiste in sessionStorage insieme ai dati dell'analisi
- **Pagina /analysis**: legge `collezioneId` da sessionStorage e lo passa ad `AnalysisResult`
- **AnalysisResult**: se `collezioneId` è presente, lo include nel FormData della fetch e dopo il salvataggio fa redirect a `/collezione/{id}` anziché mostrare il toast standard

## Strategia di Test

Si aggiornano i test API per coprire il nuovo branch (collezioneId presente vs assente) e si aggiunge un test sulla pagina dettaglio per verificare la presenza del CTA. Il flusso end-to-end attraverso i componenti client è coperto dai test unitari di ciascun layer.

- Test API: caso con `collezioneId` valido (aggiunge a collezione esistente), con `collezioneId` di un altro utente (403), con `collezioneId` inesistente (404)
- Test pagina dettaglio: verificare che il CTA "Nuova analisi" sia presente con href corretto `/?collezioneId={id}`
- Test AnalysisResult: verificare che `collezioneId` venga incluso nel FormData quando presente
