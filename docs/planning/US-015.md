## User Story

**Epic:** EP-005 — Autenticazione e Account Utente
**Priorità:** MEDIUM | **Story Points:** 2

**Story**
Come Giulia,
voglio visualizzare e modificare i miei dati personali,
così da mantenere il mio account aggiornato.

**Criteri di Accettazione**
- [ ] L'utente autenticato può visualizzare la propria email nella pagina profilo
- [ ] L'utente può cambiare la propria password inserendo quella attuale e una nuova (con conferma)
- [ ] Una password attuale errata mostra un messaggio di errore chiaro
- [ ] Un tentativo di accesso alla pagina senza login reindirizza alla pagina di login

---

## Soluzione Tecnica

La pagina profilo è un server component che legge la sessione e redirige gli utenti non autenticati. Il cambio password avviene tramite un nuovo endpoint API che verifica la password attuale prima di aggiornare l'hash nel database.

- **Pagina `/profilo`**: server component con `ottieniSessioneServer()` per verificare autenticazione e mostrare email; form client per cambio password
- **API `POST /api/auth/cambio-password`**: riceve password attuale, nuova password e conferma; verifica la password attuale con bcrypt, valida la nuova (>= 8 caratteri), aggiorna `passwordHash` nel database
- **Navbar**: aggiunta link al profilo cliccando sull'avatar/email esistente nella navbar

---

## Strategia di Test

Test focalizzati sull'endpoint API di cambio password (validazione, autorizzazione, casi di errore) e sulla protezione della pagina profilo per utenti non autenticati.

- Test API cambio password: password attuale errata → 401, nuova password troppo corta → 400, password e conferma non corrispondono → 400, cambio riuscito → 200
- Test protezione pagina: accesso senza sessione → redirect a /accesso
- Test integrazione: verifica che dopo il cambio password il login funzioni con la nuova password
