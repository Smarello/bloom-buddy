## User Story

**Epic:** EP-006 — Collezione Personale e Storico Analisi
**Priorità:** HIGH | **Story Points:** 3

**Story**
Come Giulia,
voglio salvare l'analisi di una pianta nella mia collezione,
così da poterla consultare in futuro senza dover ripetere la foto.

**Criteri di Accettazione**

- [ ] Il pulsante "Salva nella collezione" è visibile solo agli utenti autenticati
- [ ] L'analisi (foto compressa, specie identificata, diagnosi, consigli) viene salvata nel database associata all'account utente
- [ ] Un messaggio di conferma incoraggiante appare dopo il salvataggio avvenuto con successo
- [ ] Un'analisi identica (stessa foto e stesso timestamp) non può essere salvata due volte

---

## Soluzione Tecnica

La soluzione aggiunge un modello Prisma `Analisi` per persistere le analisi, un Route Handler protetto per il salvataggio, e un endpoint leggero per il check autenticazione client-side. Il pulsante di salvataggio appare condizionalmente nella pagina risultati.

- **Modello `Analisi`:** Contiene URL foto su Vercel Blob, dati analisi (JSON), hash foto (SHA-256) per deduplicazione, relazione con `User`. Il vincolo di unicità è su `(utenteId, hashFoto)` per impedire salvataggi duplicati della stessa immagine.
- **Storage foto:** La foto compressa viene caricata su **Vercel Blob** (CDN integrata) tramite `@vercel/blob`. Nel database si salva solo l'URL risultante.
- **API `POST /api/collezione`:** Valida la sessione, calcola l'hash della foto, verifica duplicati, carica la foto su Vercel Blob, salva URL e dati analisi nel database. Restituisce 409 se duplicato, 201 se salvato.
- **API `GET /api/auth/sessione`:** Endpoint leggero che restituisce `{ autenticato: true/false }` — necessario perché la pagina analisi è un client component e non può accedere direttamente alla sessione server-side.
- **Frontend:** Il componente `AnalysisResult` riceve una prop `utenteAutenticato` dalla pagina. Un pulsante "Salva nella collezione" appare sotto la hero card. Dopo il salvataggio, un messaggio inline incoraggiante conferma l'operazione e il pulsante diventa disabilitato con testo "Salvata nella collezione".

## Strategia di Test

I test coprono il flusso critico di salvataggio end-to-end: autenticazione, deduplicazione e feedback visivo.

- **Test API `/api/collezione`:** Salvataggio con successo (201), rifiuto utente non autenticato (401), rifiuto duplicato (409), validazione payload incompleto (400)
- **Test API `/api/auth/sessione`:** Risposta autenticato/non autenticato
- **Test componente:** Pulsante visibile solo quando `utenteAutenticato=true`, stato di caricamento durante il salvataggio, messaggio di conferma dopo successo, gestione errore duplicato
